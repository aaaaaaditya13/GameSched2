1. First Come First Serve (FCFS)

How it Works:

  • • Processes execute in arrival order
  • • Non-preemptive - no interruptions
  • • Simple but inefficient
  • • Causes "convoy effect"
Gaming Impact: Player input may lag behind enemy AI if enemies arrive first

What You'll See:

• All entities start at same position
• Enemies get CPU first (arrive first)
• Player COMPLETELY BLOCKED (yellow ring)
• Player can't move until ALL enemies finish
• Finally player gets CPU and moves

This is why FCFS is terrible for interactive systems!

2. Round Robin (RR)

How it Works:

  • • Each process gets fixed time slice
  • • Preemptive - processes can be interrupted
  • • Fair time sharing
  • • Good for interactive systems
Gaming Impact: Consistent but potentially delayed responses for all entities

What You'll See:

• Time Quantum = 2 seconds
• Enemy runs for exactly 2s, then FORCED to stop
• Player gets exactly 2s, then FORCED to stop
• Next enemy gets 2s, and so on...
• Watch the quantum countdown timer!

Fair but not optimal for user interaction

3. Priority Scheduling

How it Works:

  • • Higher priority processes execute first
  • • Player = Priority 1 (highest)
  • • Enemies = Priority 3 (lowest)
  • • Can be preemptive or non-preemptive
Gaming Impact: Player gets priority - much more responsive gameplay

Priority Order:

Priority 1: Player Input
Priority 2: Game Logic
Priority 3: Enemy AI

Real-World Applications

FCFS Usage

  • • Batch processing systems
  • • Print job queues
  • • Simple embedded systems

Round Robin Usage

  • • Time-sharing systems
  • • Multi-user environments
  • • Fair resource allocation

Priority Usage

  • • Real-time systems
  • • Gaming engines
  • • Critical system processes
